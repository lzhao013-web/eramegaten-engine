# eraMegaten Engine (modern Python)

这是一个面向 `E:\mgt` 中 eraMegaten 数据的现代 Python 3.11 复刻引擎骨架。它不修改原游戏目录，而是从原始 `ERB/ERH/CSV` 读取脚本和数据，在新的运行时里解析并执行 EraBasic/Emuera 风格脚本。

## 当前能力

- 读取 `ERB/*.ERB`, `*.erb`, `*.ERH`，建立函数、标签、变量声明、`#DEFINE` 索引；支持 `#DIM/#DIMS CONST/GLOBAL/SAVEDATA/CHARADATA`、声明初始化值、`#LOCALSIZE/#LOCALSSIZE`，以及用 `#DIM CONST`/`#DEFINE` 常量书写的数组维度；`#DEFINE` 可作为数值常量或变量/数组左值别名继续参与读写。
- 读取 `CSV` 基础映射、`_Rename.csv` 常量、`_Replace.csv`（含 MONEYSTR 用お金の単位/単位の位置）、`GameBase.csv`（含 `最初からいるキャラ` 初期角色）、`VariableSize.csv`、`Chara*.csv` 角色模板；角色模板同时按 CSV 内 `番号/NO`、SP/非 SP 与 `Chara*.csv` 文件编号索引，`CSVCHARANUM()` 可返回稀疏角色 CSV 编号扫描上界。
- 解释 EraBasic 核心流程：`IF/ELSEIF/ELSE/ENDIF`、`SIF`、`SELECTCASE/CASE/CASEELSE`、`FOR/NEXT`、`WHILE/WEND`、`DO/LOOP`、`REPEAT/REND`、`BREAK/CONTINUE`、`GOTO/GOTOFORM/TRYGOTO/TRYGOTOFORM/TRYCGOTO/TRYCGOTOFORM`、`JUMP/TRYJUMP/TRYCJUMP/TRYJUMPFORM/TRYCJUMPFORM/TRYJUMPLIST`、`CALL/CALLFORM/TRYCALL*/TRYCALLLIST`、`TRYGOTOLIST`、`RETURN/RETURNF/RETURNFORM`、`RESTART`、`BEGIN`；`BEGIN` 状态迁移会按私家改造版语义重置可见字体/颜色样式并丢弃旧画面的非交互按钮候选，覆盖 `SHOP/TRAIN/AFTERTRAIN/ABLUP/TURNEND` 等 eraMegaten 训练-商店状态；`GOTO`/`BREAK` 跳出内层循环时会清理已离开的循环上下文，`RESTART` 会按 Emuera 语义重启当前函数而不是全局回标题。
- 表达式：整数/字符串、数组变量、CSV 名称索引/角色 CSV 读取函数（含相関関係）、`[[...]]` 常量、允许前导零的 Era 数字、三项 `? #`、逻辑/位运算（含 `!&` NAND、`!|` NOR）、数学函数（含私家改造版 `CBRT/LOG/LOG10/EXPONENT`）、字符串重复与拼接、前/后置 `++`/`--`（含 `++(LOCAL:4) % N` 这类带括号左值的前置自增表达式），以及数组下标中的函数调用；form string 中的 `{COLOR("AQUA")}` 和 `%TOSTR(X, "00")%` 这类嵌套引号也可正确分词；`COLOR()` 原生 fast path 按 eraMegaten 的 `COLOR.ERB` 自定义颜色表返回色值，`SETCOLORBYNAME` 则按 Emuera/.NET KnownColor 名称处理裸色名。
- 表达式兼容常见 Era 写法：函数调用可带省略参数（如 `FUNC(A, , , B)`），CSV/rename 名称可包含括号（如 `FLAG:Foo(1)`）；`SUBSTRING(S, start, -1)`/`SUBSTRING S, start, -1` 按 Emuera 惯用法返回从 `start` 到末尾并写回 `RESULTS`；`STRCOUNT/REPLACE` 按私家改造版说明使用正则表达式模式，`REPLACE` 支持常见 .NET `$1` 分组替换。
- 函数头默认参数会在调用方省略对应参数时生效，覆盖 `CALL F, A, , , 4` 和表达式函数 `F(, 3)` 两种路径；没有声明默认值的省略参数仍保持 Emuera 式空字符串/数值 0 绑定。
- `_Rename.csv` 兼容数值常量与文本别名两种用法，支持 `[[...]]` 展开为数组基址后继续带下标（如 `[[店舗:種類]]:(idx)`），也支持作为赋值左值使用。
- 输出与表单字符串：`PRINT*`、`PRINTFORM*`、`PRINTS*` 字符串表达式输出、`PRINTV*` 数值/表达式输出、`PRINTBUTTON/PRINTBUTTONC/PRINTBUTTONLC`、`HTML_PRINT`、`PRINTLC/PRINTD/PRINTDL/PRINT*D*`（D 系忽略 SETCOLOR 使用默认文字色）、`FORCEKANA` 影响的 `PRINTK*`、`PRINTDATA*` 均匀随机数据块（含 `DATALIST/ENDLIST` 多行候选与候选编号回写；直接执行到 `DATA/DATAFORM/DATALIST/ENDLIST/ENDDATA` 记录时保持空操作）、`DEBUGPRINT*`（按 `デバッグコマンドを使用する` 配置默认识别为无可见输出操作，启用时写入独立 debug transcript）、`PRINT_SPACE`、`%expr%`、`{expr}`、`\@ cond ? A # B \@`；`%\@ cond ? text # \@%` 这种外层百分号包裹的 form 条件会输出文字分支而不会把分支再次当表达式求值；`\%`、`\(`/`\)`、`\n` 等 form 文本转义会在渲染后还原为字面百分号、括号或换行；`RETURNF \@...\@` 这类未加 `@"..."` 的 form 条件返回值也会先渲染再写入 `RESULTS`。
- 终端转写兼容 `REUSELASTLINE`、`CLEARLINE` 与 `SKIPDISP`：带参数的 `REUSELASTLINE` 会替换上一条已缓冲显示行，无参数时作为 `CLEARLINE` 后常见的光标复用提示保留为无输出副作用操作；`CLEARLINE` 会同步丢弃已清除行上的非交互按钮候选，避免输入校验循环误删额外行或误点已经被重绘/隐藏的旧按钮；`SKIPDISP` 会在保持脚本逻辑执行的同时抑制隐藏区输出和隐藏 `PRINTBUTTON` 候选；`NOSKIP/ENDNOSKIP` 可在不清除 `SKIPDISP` 状态的前提下临时恢复关键输出，`ISSKIP()` 可读取当前跳过显示标志。
- 普通 `PRINT/PRINTL/PRINTW` 中出现 `%...%`、`\@ ... \@` 时也会按 eraMegaten 常见 form 标记渲染，但会保留 AA/纯文本里大量出现的裸 `{...}`；`PRINTS*` 字符串表达式和 `PRINTFORM*` 仍会完整渲染 `{expr}`。`HTML_PRINT SHOW_LINE` 这类变量/表达式输出会求值，`HTML_GETPRINTEDSTR()`/`HTML_TOPLAINTEXT()` 可按当前显示行取回文本以支持动画打印辅助函数，但 HTML 按钮不会被非交互回归误当作键盘输入。非交互输入可从普通打印文本里的 `[数字]` 菜单项提取默认选择；若提供过显式 `--inputs`，输入耗尽后会停在下一次提示处，`CALL INPUTINT/INPUT_YN/INPUT_MANY/INPUT_SELECT/INPUT_SPLIT/INPUT_CHAR/INPUT_ONEKEY_TAP` 这类 eraMegaten 输入包装器也会遵循同一阻塞语义并保留调用栈/当前 PC 以便 `queue_input()` 后续跑，`CALL TINPUTINT(timeout, default, ...)` 则会按限时输入的默认值继续；当下一条显式输入与刚渲染的小型数字菜单不匹配时也会停在该提示，避免自动回归把无效值喂进脚本校验循环后空转到步数上限。
- 字符串赋值兼容 eraMegaten 常用的裸文本标签写法；即使裸文本同时出现在 `_Rename.csv`/CSV 常量中（如商店命令名），也会在字符串上下文保留为文本；`TSTR` 按 Emuera 临时字符串数组处理并读取 `CSV/TSTR.csv` 的名称索引；混合文本与 `%expr%`/`{expr}`/`\@...\@` 的 RHS、`>>> %RESULTS%` 这类带箭头前缀的战斗目标标签、`AUTO_SPLIT(...) + \@TEMP >= 0 ?+#\@ + TOSTR(TEMP)` 这类字符串表达式内联 form 片段、以及 `<font color='%X%'>` 这类 HTML tag 片段会按 form/string 语义渲染，不会把内部 `? #`、显示用括号、箭头或标签尖括号误判为顶层表达式，同时仍保留 `LOCAL`、`RESULTS`、`A / 2`、`8 >> 1`、顶层三项表达式、`F() + @"%TOSTR(X)%"` 等真实表达式求值；变量名以 `PRINT` 开头（如 `PRINT_MODE`）也不会被误识别为输出命令。
- 赋值左值兼容更多 eraMegaten 写法：全角数字可作为标识符片段（如 `１moreフラグ`），`[[...]]` 别名可作为数组左值，个别脚本中的 `FLAG:name,-= value` 尾随逗号写法也可执行。
- 支持常用 `#DIM/#DIMS REF` 数组参数别名与对应 `VARSIZE`，表达式函数里的 REF 参数也会别名到调用方数组；带下标传入的 REF（如 `CALL F(A:2, S:1)`、`CALL F(CFLAG:TARGET)`）会按前缀/偏移映射读写调用方切片，`VARSET` 与 `VARSIZE` 也按剩余切片尺寸工作；`VARSIZE(name, dim)` 可按 0 起算维度返回指定维长度，`VARSIZE` 命令会填充 `RESULT:0..n` 维度且带第二参数时把 `RESULT` 设为指定维长度；`VariableSize.csv` 的一维/二维/三维系统配置（如 `R,1000`、`SELECTCOM,1000`、`DA,100,200`、`TA,2500,20,20`）会完整保留，且这些系统变量名会优先于 CSV/_Rename 同名常量解析；`A:0 = X, Y` 这类连续下标赋值可用，使选项页/鼠标 UI 这类通过 REF 填充/打印二维表的脚本能复用调用方数组；`VARSET LOCAL, -1`、`VARSET Q, -1`、`#DIM/#DIMS` 局部数组填充值会按数组尺寸物化，`VARSET TCVAR:ARG:0,0` 与 `VARSET CFLAG:TARGET:0,0` 这类带下标目标则只清理/填充对应前缀的末维后缀，不会误清空整张表；`ARRAYCOPY/ARRAYREMOVE/ARRAYSHIFT/ARRAYSORT` 与 `SUMARRAY/MAXARRAY/MINARRAY/INRANGEARRAY/MATCH/FINDELEMENT/FINDLASTELEMENT` 等查询函数在角色数组和 REF 切片别名上会按显式或省略前缀解析。
- 裸变量与零号槽按 Emuera 数组语义互为别名：`A`/`A:0`、`RESULT`/`RESULT:0`、`LOCALS`/`LOCALS:0` 以及角色数组的 `CFLAG:chara`/`CFLAG:chara:0` 会同步读写；Emuera 固定角色数组命名空间覆盖 `SOURCE/EX/JUEL/RELATION/EQUIP/TEQUIP/STAIN/GOTJUEL/NOWEX` 等尾段变量，读写 `STAIN:chara:slot` 这类状态不会落到全局表；`#DIM/#DIMS CHARADATA` 自定义角色变量也存入每个 `CharacterState`，如 `RPG_CFLAG_DUNGEON:chara:slot`、`RPG_CSTR_DUNGEON:chara:slot` 会随角色表读写而非落入全局数组；`TALENT:親愛`、`BASE:LV`、`CSTR:一人称` 等省略角色位写法会按当前 `TARGET` 解析，`TALENT:chara:親愛` 等两段写法仍显式指定角色；函数头里的标量参数 `@FUNC(ARG, ARGS)` 绑定时也会同步填充 `ARG:0`/`ARGS:0`，因此 `INPUT_SELECT` 这类同时遍历 `ARG:n`/`ARGS:n` 的脚本可正确显示第 0 项。
- 数组/集合判定函数按 eraMegaten 用法执行：`MATCH(array, value[, start[, end]])` 会保留第一个参数为数组引用并统计切片命中数，支持字符串数组与 `ABL:ARG:0` 这类前缀扫描；`GROUPMATCH(target, ...)` 返回匹配个数，`EQUALCHECK` 保留项目脚本对尾随默认 0 参数的处理。
- `BEGIN SHOP`/读档后的商店状态会执行 `SHOW_SHOP → INPUT → USERSHOP → EVENTSHOP` 循环；`BEGIN TRAIN` 会执行 `EVENTTRAIN → SHOW_STATUS → SHOW_USERCOM → INPUT → USERCOM` 菜单循环，`DOTRAIN` 会按 eraMegaten 的共通调教分派执行 `EVENTCOM → COM_COMMON(SELECTCOM) → SOURCE_CHECK → EVENTCOMEND`；`Train.csv`/`Item.csv` 会同步填充 `TRAINNAME`/`ITEMNAME` 字符串数组，让调教菜单和物品/衣装 UI 使用真实名称；`BEGIN AFTERTRAIN` 会进入 `EVENTEND`，`BEGIN ABLUP` 会按 Emuera 内置阶段语义在可交互/有输入时执行 `ABL_MANUAL_MAIN`，随后进入真实 `EVENTTURNEND`；非交互回归在没有预置输入时停在已渲染菜单，不会自动误点状态区按钮。
- 针对 `E:\mgt` 读档后商店入口的重型表达式函数/线性扫描补了等价原生热路径（如 `ANALYZE_COUNT_CHECK`、魔晶/改造装备判定、改造装备编号映射、`装備強化_展開` 无强化数据直通分支、MouseUIStore 稀疏缓存构建、`ENTRY_EQUIPMENT_COMPENDIUM` 稀疏物品图鉴登记、`PRINT_FORMATION_FACE_P` 編成面板图片名查询、普通攻撃 `SKILL_SPECIAL_TARGET_0` 默认目标判定、战斗结束 `TEMP_STATUS_RESET` 临时补正清理），避免在 `SHOW_SHOP`、编成/闘技場队伍面板、商店鼠标 UI、普通攻击选目标路径与胜利结算路径中耗尽同步函数/主循环步数预算或泄漏旧 `RESULT`。
- 兼容 eraMegaten 迷宮/队伍/商店 UI 的关键惯用法：`TOSTR(X,"00")` 零填充，以及 `TOSTR` 对 `#,###`/`#,##0` 千分位、`x`/`X8` 十六进制、`"0'.'00"` 小数点插入等常用格式；DPOINT 当前迷宮简写、字符串 `+=` 中的 form 条件与表达式拼接、`%expr%` 形式的字符串赋值、字符串数组 `FINDELEMENT/FINDLASTELEMENT`（含 start/end 范围、正则部分匹配与 completeMatch 完全匹配参数）、表达式函数调用时保护外层 `RESULT/RESULTS`，且 `#FUNCTION` 无 `RETURNF` 落空时返回中性默认值。

## 快速使用

```powershell
# 盘点 E:\mgt
python -m eramegaten_engine.cli audit E:\mgt --top 30

# 查看函数
python -m eramegaten_engine.cli inspect E:\mgt SYSTEM_TITLE --limit 80

# 运行标题入口；--inputs 可预置输入，--non-interactive 避免阻塞
# --state-dir 指定 Python 引擎 sidecar 存档目录，避免写入 E:\mgt
python -m eramegaten_engine.cli run E:\mgt --entry SYSTEM_TITLE --inputs 1,0,99 --non-interactive --max-steps 20000 --state-dir .\state

# 执行后导出指定 runtime graphic 或资源/运行时 sprite 为 PNG（可重复指定）
python -m eramegaten_engine.cli run E:\mgt --entry SOME_GRAPHIC_PROBE --non-interactive --quiet --export-graphic 8101=.\exports\canvas.png --export-sprite A1_0_1=.\exports\face.png

# 显式导出 Emuera 1808 二进制快照到指定路径；默认不会覆盖已有文件
python -m eramegaten_engine.cli run E:\mgt --entry TITLE_LOADGAME --inputs 0 --non-interactive --quiet --export-native-save .\exports\probe.sav --native-save-text "probe"
```
